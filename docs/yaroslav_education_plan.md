# 🎓 Ярослав — Система обучения и развития ребёнка

> **Ребёнок:** Ярослав, 5 лет  
> **Цель:** Выстроить систему обучения, развития и эмоциональной поддержки через AI  
> **Архитектура:** Секция «👶 Ярослав» в родительском боте + отдельный бот для ребёнка

---

## Концепция

Две точки входа:

1. **Для родителей** — секция «👶 Ярослав» в едином боте (`@lifeos_life_bot`)
   - Расписание занятий и активностей
   - Трекинг прогресса (буквы, цифры, навыки)
   - Дневник развития (заметки, milestones)
   - Управление напоминаниями и контентом

2. **Для Ярослава** — отдельный бот (`@yaroslav_life_bot` или аналог)
   - AI-друг с детским, тёплым голосом (TTS — аудиосообщения)
   - Обучающие мини-сессии (буквы, цифры, логика)
   - Физическая активность (зарядка, приседания)
   - Эмоциональная поддержка (психолог для ребёнка)
   - Сказки и истории на ночь
   - Простые игры (угадай букву, посчитай предметы)

## Архитектура

### Родительская секция (unified bot)

```

├── __init__.py
```

**Кнопки для родителей:**

| Кнопка | Что делает |
|--------|-----------|
| 📅 **Расписание** | Недельное расписание занятий (буквы Пн/Ср/Пт, цифры Вт/Чт, зарядка ежедневно) |
| 📊 **Прогресс** | Какие буквы выучил, какие цифры знает, физ. активность |
| 📝 **Дневник** | Заметки о развитии: «Сегодня впервые написал букву А» |
| ⚙️ **Настройки** | Время занятий, список тем, голос AI, возрастные ограничения |
| 💬 **Диалоги** | Просмотр переписки ребёнка с AI (фильтрация по дате) |
| 📚 **Программа** | Учебный план: какие буквы/цифры в каком порядке |

**Доступ:** admin (Алексей) + wife (жена) — оба родителя.

### Детский бот (отдельный)

```
src/bots/child/
├── __init__.py
├── handlers.py      # Хэндлеры для ребёнка
├── keyboard.py      # Детские кнопки (большие, с эмодзи)
├── prompts.py       # Промпт AI-друга для ребёнка
├── scheduler.py     # Напоминания ребёнку (аудио)
├── curriculum.py    # Учебная программа: буквы, цифры, навыки
└── games.py         # Мини-игры: угадай букву, посчитай, и т.д.
```

**Кнопки для ребёнка (крупные, понятные):**

| Кнопка | Что делает |
|--------|-----------|
| 📖 **Учить буквы** | AI показывает букву, произносит, просит повторить |
| 🔢 **Считать** | Считаем предметы, учим цифры |
| 🏃 **Зарядка** | AI озвучивает упражнения (аудио): «Давай 5 приседаний!» |
| 🎨 **Игра** | Мини-игра: загадка, найди лишнее, рифма |
| 📖 **Сказка** | AI рассказывает сказку (аудио) — перед сном или по запросу |
| 💬 **Поговорить** | Свободный разговор с AI-другом |

**Почему отдельный бот:**
- Ребёнок не должен видеть финансы, бизнес, медицинские данные
- Другой UX: большие кнопки, всё через аудио, минимум текста
- Отдельный токен → отдельный контекст и ограничения
- Можно дать ребёнку телефон/планшет с одним ботом

## База данных

### Новые таблицы

```sql
-- Учебная программа (что учим и в каком порядке)
CREATE TABLE child_curriculum (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    child_user_id BIGINT NOT NULL REFERENCES users(user_id),
    subject TEXT NOT NULL,        -- 'letters', 'numbers', 'physical', 'logic'
    item TEXT NOT NULL,           -- 'А', 'Б', '1', '2', 'приседания'
    status TEXT DEFAULT 'pending', -- 'pending', 'learning', 'mastered'
    started_at TIMESTAMPTZ,
    mastered_at TIMESTAMPTZ,
    attempts INT DEFAULT 0,       -- сколько раз практиковал
    success_rate REAL DEFAULT 0,  -- процент правильных ответов
    sort_order INT DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT now()
);

-- Сессии обучения (каждый контакт ребёнка с ботом)
CREATE TABLE child_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    child_user_id BIGINT NOT NULL REFERENCES users(user_id),
    session_type TEXT NOT NULL,   -- 'letters', 'numbers', 'physical', 'game', 'talk', 'story'
    started_at TIMESTAMPTZ DEFAULT now(),
    ended_at TIMESTAMPTZ,
    duration_minutes INT,
    items_practiced JSONB,        -- ["А", "Б", "В"] — что практиковали
    correct_answers INT DEFAULT 0,
    total_questions INT DEFAULT 0,
    mood TEXT,                    -- 'happy', 'neutral', 'sad', 'angry' — настроение ребёнка
    notes TEXT                    -- AI summary сессии
);

-- Расписание занятий
CREATE TABLE child_schedule (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    child_user_id BIGINT NOT NULL REFERENCES users(user_id),
    day_of_week INT NOT NULL,     -- 0=Пн, 6=Вс
    time_slot TIME NOT NULL,      -- 10:00, 15:00
    activity TEXT NOT NULL,       -- 'letters', 'numbers', 'physical', 'reading'
    duration_minutes INT DEFAULT 15,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT now()
);

-- Дневник развития (записи родителей)
CREATE TABLE child_diary (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    child_user_id BIGINT NOT NULL REFERENCES users(user_id),
    parent_user_id BIGINT NOT NULL REFERENCES users(user_id),
    entry_text TEXT NOT NULL,
    milestone BOOLEAN DEFAULT false,  -- ключевое достижение
    tags TEXT[],                      -- ['буквы', 'первый раз', 'прорыв']
    created_at TIMESTAMPTZ DEFAULT now()
);

-- Индекс для быстрого поиска
CREATE INDEX idx_child_sessions_user ON child_sessions(child_user_id);
CREATE INDEX idx_child_curriculum_user ON child_curriculum(child_user_id, subject);
CREATE INDEX idx_child_schedule_user ON child_schedule(child_user_id, day_of_week);
```

## AI-Промпт для детского бота

```
Ты — Добрый Друг Ярослава. Тебя зовут [имя на выбор: Умка / Знайка / Совёнок].

ПРАВИЛА:
- Ярославу 5 лет. Говори ПРОСТО: короткие предложения, понятные слова
- Будь ТЁПЛЫМ и ПОДДЕРЖИВАЮЩИМ. Хвали за каждую попытку
- НИКОГДА не критикуй, не ругай, не пугай
- Используй эмодзи в тексте, но основной канал — ГОЛОС (аудио)
- Максимум 2-3 предложения за раз
- Если ребёнок грустит/злится — выслушай, поддержи, НЕ обесценивай чувства
- НЕ упоминай: смерть, насилие, страшные темы, политику, религию
- Если ребёнок спрашивает что-то неуместное — мягко переведи тему
- Всегда заканчивай на позитивной ноте

ОБУЧЕНИЕ:
- Буквы: показывай одну букву, назови слово на эту букву, попроси придумать своё
- Цифры: считай предметы, простые примеры (1+1, 2+3)
- Зарядка: называй упражнение, считай вслух повторения
- Если ребёнок устал — предложи сказку или игру, не настаивай

КОНТЕКСТ:
{current_time} — текущее время
{today_schedule} — расписание на сегодня
{current_subject} — текущий предмет (буквы/цифры/зарядка)
{progress} — прогресс: какие буквы уже знает, какие цифры
{last_sessions} — последние 3 сессии (что делали, как прошло)
{mood_history} — настроение за последние дни
```

## Система напоминаний (scheduler.py)

### Для ребёнка (аудиосообщения через TTS)

| Время | Сообщение | Условие |
|-------|-----------|---------|
| 09:30 | «Доброе утро, Ярослав! Давай сделаем зарядку? 🏃» | Ежедневно |
| 10:00 | «Пора учить буквы! Сегодня у нас буква {next_letter}» | Пн, Ср, Пт |
| 10:00 | «Давай посчитаем! Сегодня учим цифру {next_number}» | Вт, Чт |
| 15:00 | «Время для игры! Хочешь загадку? 🎨» | Ежедневно |
| 20:00 | «Хочешь сказку на ночь? 📖» | Ежедневно |

- Все сообщения отправляются как **аудио** (TTS → voice message)
- Inline-кнопки: «✅ Давай!» / «⏰ Позже» / «❌ Не сегодня»
- Если ребёнок нажал «Позже» — повторить через 30 мин (макс. 1 повтор)
- Если «Не сегодня» — пропустить, записать в лог

### Для родителей (текстовые отчёты)

| Время | Сообщение |
|-------|-----------|
| 21:00 (ежедневно) | Сводка дня: сколько занятий, что выучил, настроение |
| Воскресенье 20:00 | Недельный отчёт: прогресс, рекомендации, достижения |
| При milestone | «🎉 Ярослав выучил все буквы от А до Ж!» |

### Прогрессия

- **1 буква за 2-3 дня** (повторение → новая)
- **1 цифра за 2-3 дня**
- **Зарядка ежедневно** с постепенным усложнением
- AI адаптирует темп: если ребёнок отвечает правильно 3 раза → переход к следующей
- Если ошибается → больше повторений, другой подход (слово-картинка-звук)

## Безопасность

### Контентная фильтрация
- **Строгий system prompt** — AI не может выйти за рамки разрешённых тем
- **Запрещённые темы:** насилие, смерть, страх, политика, религия, сексуальность
- **Запрет на манипуляции:** AI не может просить ребёнка делать что-то без ведома родителей

### Временные ограничения
- Бот активен **08:00—21:00** (настраивается родителями)
- После 21:00: «Пора спать, Ярослав! Увидимся завтра утром 🌙»
- Максимум **30 минут подряд** на одну сессию → «Давай отдохнём! Возвращайся через 15 минут»

### Родительский контроль
- Все диалоги сохраняются и доступны родителям через секцию «💬 Диалоги»
- Родители могут менять расписание, темы, голос AI
- Уведомление родителям, если ребёнок в плохом настроении 3+ дня подряд
- Родитель может «заблокировать» бота на время (через настройки)

### Данные
- `child_user_id` изолирован — ребёнок не видит данные родителей
- Все запросы от ребёнка обрабатываются через специальный детский промпт
- Логирование всех сессий для анализа

## Эмоциональная поддержка (детский психолог)

Когда ребёнок пишет/говорит что-то эмоциональное:

**Триггеры:** «мне грустно», «я злюсь», «мама ругается», «не хочу», плач (аудио)

**AI реагирует:**
1. Валидация чувства: «Я понимаю, что тебе сейчас грустно. Это нормально.»
2. Активное слушание: «Расскажи, что случилось?»
3. Поддержка: «Ты молодец, что рассказал мне. Давай вместе подумаем.»
4. Переключение (если готов): «Хочешь, я расскажу тебе сказку?»

**Правила:**
- НЕ обесценивать: ❌ «Не плачь», «Это ерунда» → ✅ «Я понимаю, что тебе обидно»
- НЕ вставать на чью-то сторону в конфликтах с родителями
- При тревожных сигналах → немедленное уведомление родителям
- Тревожные сигналы: упоминание насилия, самоповреждения, страха перед взрослыми
