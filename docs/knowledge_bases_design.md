# Базы знаний — Доктор и Бизнес

## Текущая инфраструктура

- **Obsidian vault** синхронизируется через Syncthing (VPS ↔ ПК)
- **Папка `02-Knowledge/`** уже индексируется RAG-системой (watcher.py → embeddings → pgvector)
- **Модель эмбеддингов:** text-embedding-3-small (1536 dim)
- **Поиск:** cosine similarity по `events.embedding`, фильтрация по `user_id`
- **Новые заметки автоматически индексируются** — watcher.py отслеживает изменения в vault

→ Для запуска баз знаний **не нужны изменения в коде** — достаточно создать структуру папок и шаблоны заметок.

### Как это работает

1. Ты заполняешь заметку в Obsidian (или фото анализа → через бота → автоэкспорт)
2. Watcher.py подхватывает изменения → создаёт эмбеддинг → сохраняет в pgvector
3. В режиме 🩺 Доктор спрашиваешь: «Когда последний раз сдавал кровь?» → RAG находит заметку
4. AI-доктор видит результаты и может сравнить с предыдущими

### Будущее улучшение (Фаза N+1)

- **Парсинг фото анализов**: отправляешь фото → Vision AI извлекает таблицу → автоматически создаёт заметку
- **Напоминания**: «Прошло 6 месяцев с последнего анализа крови» (APScheduler)
- **Тренды**: графики изменений показателей (гемоглобин за год)

### Как это работает

1. Заполняешь заметки в Obsidian на ПК → синхронизация через Syncthing
2. RAG автоматически индексирует все заметки из `02-Knowledge/Бизнес/`
3. В секции 💼 Бизнес спрашиваешь: «Какой прайс на уборку после ремонта?» → RAG находит заметку
4. AI-ассистент отвечает на основе твоих данных

### Изоляция проектов

RAG ищет по всем заметкам пользователя. Для точности можно:
- Указывать проект в запросе: «Прайс КДК» vs «Стратегия SEO»
- Использовать теги в frontmatter для дополнительной фильтрации (будущее улучшение)
- Связать заметки с `project_id` из БД (потребует доработки watcher.py)
