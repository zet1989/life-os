# Базы знаний — Доктор, Бизнес, SEO

## Текущая инфраструктура

- **Obsidian vault** синхронизируется через Syncthing (VPS ↔ ПК)
- **Папка `02-Knowledge/`** уже индексируется RAG-системой (watcher.py → embeddings → pgvector)
- **Модель эмбеддингов:** text-embedding-3-small (1536 dim)
- **Поиск:** cosine similarity по `events.embedding`, фильтрация по `user_id`
- **Новые заметки автоматически индексируются** — watcher.py отслеживает изменения в vault

→ Для запуска баз знаний **не нужны изменения в коде** — достаточно создать структуру папок и шаблоны заметок.

## ✅ Чанкинг больших документов (реализовано)

### Как работает

Watcher.py автоматически разбивает большие файлы на чанки:

- **Размер чанка:** ~4000 символов (~2-3 страницы)
- **Перекрытие:** 400 символов (контекст между чанками не теряется)
- **Умное разделение:** разрезает по заголовкам Markdown (`##`, `###`) или по абзацам, а не посередине предложения
- **Один файл → несколько векторов** в pgvector, каждый с метаданными `chunk_index` и `total_chunks`
- **Маленькие файлы (< 4000 символов)** — работают как раньше, один вектор
- **Обновление:** при изменении файла пересчитываются только изменённые чанки, лишние удаляются

### Примеры

| Документ | Размер | Чанков | Покрытие |
|----------|--------|--------|----------|
| Короткая заметка (2 стр.) | 3 000 символов | 1 | 100% |
| Гайд по SEO (30 стр.) | 50 000 символов | ~14 | 100% |
| Книга (200 стр.) | 400 000 символов | ~110 | 100% |

### Как использовать

1. Положи `.md` файл любого размера в `02-Knowledge/` (или другую RAG-папку)
2. Syncthing синхронизирует на VPS
3. Watcher.py автоматически разобьёт на чанки и проиндексирует каждый
4. RAG найдёт нужный фрагмент через семантический поиск

Пример: книга «SEO для бизнеса» (200 стр.) → разбить на:
- `Глава 1 — Основы SEO.md`
- `Глава 2 — Ключевые слова.md`
- `Глава 3 — Линкбилдинг.md`
- ...и т.д.

### Как это работает

1. Ты заполняешь заметку в Obsidian (или фото анализа → через бота → автоэкспорт)
2. Watcher.py подхватывает изменения → создаёт эмбеддинг → сохраняет в pgvector
3. В режиме 🩺 Доктор спрашиваешь: «Когда последний раз сдавал кровь?» → RAG находит заметку
4. AI-доктор видит результаты и может сравнить с предыдущими

### Как хранятся анализы: фото vs текст

**Оба формата — но текст главный:**

1. **Фото анализа** → отправляешь в режиме 🩺 Доктор
2. **Vision AI** расшифровывает: извлекает таблицу показателей, значения, нормы
3. **Текстовая расшифровка** сохраняется в `events` (тип `doctor`) + AI даёт интерпретацию
4. **Автоэкспорт в Obsidian** (будущее): создаёт `.md` заметку в `02-Knowledge/Здоровье/Анализы/` по шаблону выше
5. **Оригинальное фото** архивируется на диск (media archive)

→ RAG ищет по **текстовой расшифровке**, не по фото. Это надёжнее и быстрее.

**Текущий flow (работает уже сейчас):**
- Фото анализа → Vision AI → текстовый ответ → сохранение в events → RAG-доступен

**Будущий flow (доработка writer.py):**
- Фото анализа → Vision AI → структурированная Markdown-заметка → Obsidian → RAG

### Будущее улучшение (Фаза N+1)

- **Автоэкспорт анализов**: Vision AI → парсинг таблицы → `.md` заметка в Obsidian автоматически
- **Напоминания**: «Прошло 6 месяцев с последнего анализа крови» (APScheduler)
- **Тренды**: графики изменений показателей (гемоглобин за год)
- **PDF-хранилище**: привязка PDF/фото к заметке через `![[файл]]`
